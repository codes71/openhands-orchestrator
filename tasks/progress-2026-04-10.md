## What's NOT Working Yet

### Implementation Pipeline — BLOCKED
The implementation workflow (comment on research issue → create issue in target repo → OpenHands resolves → PR) fails because:

1. **Fine-grained PAT lacks permissions** — OpenHands resolver needs a **classic PAT** with `repo` + `workflow` scopes. The current fine-grained PAT (`github_pat_11AVNQ3YY...`) can create issues but can't do everything OpenHands needs (got "Resource not accessible by personal access token").

2. **Research workflow label application** — the `gh issue create --label` command in research.yml doesn't reliably apply labels. The PAT may need `issues:write` scope explicitly. Labels had to be added manually for the first run.

## Key Learnings

1. **Claude Code OAuth tokens rotate** — they expire within ~1 hour. Use standard API keys (`sk-ant-api03-...`) for GitHub Actions.
2. **Fine-grained PATs have limited scope** — OpenHands needs a classic PAT with `repo` + `workflow`.
3. **OpenHands org moved** — `All-Hands-AI/OpenHands` → `OpenHands/OpenHands`. Reusable workflow refs don't follow redirects.
4. **Self-hosted runners skip `actions/setup-python`** — use system Python directly.
5. **Rate limits on Claude Code sub** — the orchestrator shares rate limits with active Claude Code sessions. Retry logic is essential.
